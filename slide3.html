Lógica, Algoritmos e Python

O Seu Papel nesta Aula

O foco NÃO é transformar você em um programador.

Em 2026, a IA escreve a maior parte do código. O papel do gestor hoje é analisar os dados:

  • Auditar: Entender o que a IA gerou.
  • Validar: Conferir se a lógica de negócio está correta.
  • Comunicar: Falar a mesma língua das equipes técnicas.

O que é Lógica?

É a organização do pensamento para atingir um objetivo. No dia a dia, você já usa lógica:

Exemplo no RH:

SE o colaborador tem mais de 1 ano de casa E não tirou férias, ENTÃO emita o aviso.

A lógica de programação é apenas a tradução desse raciocínio para uma regra que a máquina entenda.

O Algoritmo

Um algoritmo é uma sequência finita de passos para resolver um problema.

Uma receita de bolo é um algoritmo.

O GPS calculando a rota é um algoritmo.

Se o passo a passo estiver errado, o resultado será errado, não importa quão rápida seja a máquina.

Algoritmos de Negócio

Quase todos os processos de uma empresa podem ser desenhados como algoritmos:

  1. Receber nota fiscal.
  2. Verificar se o valor está no orçamento.
  3. Se sim, enviar para pagamento.
  4. Se não, devolver ao departamento.

Programar é apenas automatizar esses passos usando uma linguagem como Python.

Sintaxe

1. Variáveis

Espaços na memória para guardar informações.

setor = "Produção"
eficiencia = 94.5
meta_atingida = True

Como gestor, você revisa se os nomes fazem sentido para o negócio.

Sintaxe

2. Tipos de Dados

A IA precisa saber que tipo de "caixa" está usando:

  • str (String): Textos e nomes.
  • int (Integer): Números inteiros (contagem).
  • float: Números decimais (valores, pesos).
  • bool (Boolean): Decisões (Verdadeiro/Falso).
Lógica

3. Condicionais (if / else)

É aqui que o código "toma decisões" baseadas em regras.

if estoque < 10:
    print("Gerar ordem de compra")
else:
    print("Estoque ok")

O que o gestor olha aqui? Se o limite (10) está correto conforme a estratégia.

Estrutura

4. Listas [ ]

Agrupam vários itens similares.

maquinas_paradas = ["Torno-01", "Fresa-04", "Prensa-02"]

Útil para quando a IA precisa processar uma fila de tarefas ou uma lista de e-mails.

Estrutura

5. Dicionários { }

Dados organizados com rótulos (Chave: Valor).

produto = {
  "sku": "AB-123",
  "preco": 150.00,
  "fornecedor": "Metalúrgica X"
}

É assim que a IA organiza fichas cadastrais de forma eficiente.

Escala

6. Repetição (for)

Garante produtividade. Faz a mesma tarefa mil vezes sem cansar.

for funcionario in lista_rh:
    enviar_contracheque(funcionario)

Para o gestor, isso significa padronização e redução de erro humano.

Padronização

7. Funções (def)

Um conjunto de instruções que você "apelida" para usar depois.

def calcular_imposto(valor):
    return valor * 0.15

Se a alíquota mudar de 15% para 18%, você só precisa validar a alteração em um único lugar no código.

Como ler o que a IA gera?

Ao pedir um script para a IA, procure por:

  • Comentários: Linhas que começam com # (explicam o que o código faz).
  • Nomes de Variáveis: Devem ser claros (ex: `custo_total` em vez de `x`).
  • Lógica Central: O `if` está decidindo o que você pediu?

Atenção à Indentação

No Python, o espaço vazio no início da linha diz quem manda em quem.

Correto:

if True:
    fazer_algo()

Errado:

if True:
fazer_algo()

Se a IA gerar um código que não roda, 90% das vezes é falta de um espaço.

Checklist

  • Algoritmo: É o meu processo desenhado.
  • Variável: É onde guardo o dado.
  • If/Else: É onde decido o caminho.
  • Loop: É onde ganho velocidade.
  • Função: É o meu padrão de trabalho.
